# Contributing Guidelines

## Principles
- Keep changes small and focused
- Prefer clarity over cleverness
- Document decisions

## AI Usage
AI tools may be used for:
- Planning
- Drafting code
- Writing tests
- Documentation

All AI-generated output must be:
- Reviewed by a human
- Understood before merging

## AI Responsibilities

AI may assist with:
- Planning
- Code drafting
- Test generation
- Documentation

AI should not:
- Approve PRs
- Be the sole reviewer of its own code
- Decide release readiness
- Generate final commit messages without human review

## Pull Requests
- Explain why the change exists
- Describe risks
- Note testing performed

---

Checklist exceptions are allowed but must be explicit and documented.
